Changelog for Mark It Down features

Version history and release notes.

v2.2.10New reading surfaces and a richer knowledge graph.Under Review↓

This release adds two entirely new reading surfaces and reworks how notes connect. GitHub Repository Reader lets you browse repositories without leaving Mark It Down: navigate the file tree, follow links between files, and import what you want into your notes. RSS Reader grows from a small modal into a full-screen workspace with the same Markdown rendering quality as your own notes. The note graph replaces the shifting force-directed layout with a consistent outline tree and an interactive radial view with zoom and pan.

New Features

  • GitHub Repository Reader Browse any public GitHub repository without leaving Mark It Down. Open a repository by URL, navigate the file tree, and view Markdown files with full rendering: code blocks, diagrams, math, callouts, and images. Pin a repository so it reopens next time. Private repositories work with a personal access token. Any file can be imported into your Inbox with one click.
  • RSS Reader becomes a full-screen workspace A left feed and filter sidebar, a central article list and reading pane, and a right inspector showing metadata, tags, and headings. Filter by tag, search inline, dismiss articles with Undo, and save articles to Inbox in one step. Ctrl+Shift+R opens the surface directly.
  • Note graph: outline tree and radial view The outline view shows your notes as a navigable tree you can expand, collapse, and re-root. The radial view arranges linked notes around a center node with wheel zoom, drag pan, and a Reset button. A graph view is also available inside Repository Reader, showing how files in a repository link to each other.
  • Frontmatter editor modal Editing note metadata now opens a dedicated modal. Suggested values from the note itself and from the clipped page appear as chips you can click to apply. Copy as YAML and Clear all are available from the header menu.

Improvements

  • RSS articles render like your notes Article previews use the same rendering engine as the main editor: the same fonts, heading styles, code blocks, and Markdown formatting.
  • RSS article inspector The right panel shows the article's heading structure as a table of contents with scroll tracking. Click a heading to jump to that section.
  • Repository Reader: wikilinks and Obsidian vaults Wikilinks between files in the same repository work as navigation. Obsidian highlights, comments, and callout types render with their full styling, including collapsible sections.
  • Repository Reader: navigation history Back and Forward buttons in the header, Alt+Left / Alt+Right shortcuts, and a heading-level quick switcher (Ctrl+K) for jumping to any file by name.
  • Repository Reader: backlinks panel The right panel lists other files that link to the one you are reading, with hover previews. Click to navigate.
  • Quieter sidebar rows Status details (unprocessed marker, Git pending state, note size) move from every note row into a hover card. Git pending is indicated only by a subtle title color change.
  • Japanese text wraps naturally Japanese UI text (sidebar titles, modal labels, tooltips) now wraps at natural phrase boundaries rather than mid-word.
  • Filename becomes the note title on import Markdown files without a title field or top-level heading now use their filename as the note title, preventing a wave of untitled notes after a bulk import.

Fixed

  • Large Markdown tables Files made up entirely of large tables now open correctly instead of appearing blank or truncated.
  • Appearance panel during Focus Mode The Appearance and theme panel can now be opened and used while Focus Mode is active.
  • Frontmatter with leading blank lines Files with blank lines or whitespace before the opening frontmatter block now import their metadata correctly.
  • Git sync with spaces in filenames Pulling from Git now correctly matches notes whose filenames contain spaces, instead of creating duplicate notes.
v2.2.9OKF bundles carry their references forward, and the knowledge graph becomes easier to explore.Released↓

Knowledge stays connected when it leaves the editor. OKF bundles now carry their sources as a numbered reference list, and links between notes resolve correctly inside the exported bundle. The concept graph got a round of usability work: labels that don't overlap, amber highlights on selection, a persistent Reset button, and first-time guidance. The Portability Hub adds heading-structure warnings, and the frontmatter panel suggests standard values for confidence and source fields.

Improvements

  • OKF bundle references External links in your notes are automatically collected and appended as a numbered References list when you export as an OKF bundle. Sources stay visible when you hand off the package.
  • Note links in bundles Links between your own notes now resolve correctly inside an exported OKF bundle. Connections that existed in the editor stay intact after export.
  • OKF export type field guidance The export panel now explains how the type field is assigned automatically based on which folder a note lives in. The Portability Hub flags notes with no type set so you can review them before export.
  • Link graph improvements Node labels appear below each node and no longer overlap. Selecting a node or edge highlights it in amber. A Reset view button is always visible and returns the graph to a full overview. All four color themes now carry through into the graph correctly.
  • Concept graph new-user guidance If your graph has only one note or none at all, a short guide explains how to create connections using the [[note name]] syntax. A one-time hint also explains that clicking a node opens the table of contents for that note.
  • Note log raw status tooltip Hovering over the raw chip in the note activity log now shows a tooltip that explains what raw status means and how to clear it.
  • Portability Hub heading-level warnings The Hub now flags notes where heading levels jump unexpectedly — helping you catch structural problems before exporting to EPUB or document converters.
  • Frontmatter value suggestions When editing the confidence or source fields in the metadata panel, a short list of standard values appears. You can still type anything — the suggestions keep values consistent across notes.

Fixed

  • Toolbar menu collapsing while open Opening a toolbar menu and keeping it open no longer caused the menu to collapse unexpectedly due to the editor's auto-hide behavior.

v2.2.5 cleaned up what goes out. v2.2.6 makes it worth taking out: diagrams you can inspect at full resolution, notes you can grow by splitting, clips you can actually read. The tools for shaping information — zoom, divide, convert — now feel as capable as the tools for bringing it in.

New Features

  • Open notes from Chrome's address bar Type mid and press Space in the address bar to search notes by title. Press Enter to open in a new tab — keyboard only, no mouse required.
  • Today template Place a note named "Today" in the Template folder and every new daily note starts from that layout. One note, applied automatically each day.

Fixed

  • Web Clipper: footnote symbols in note titles Footnote reference markers were appearing in note titles and exported filenames when clipping pages with citations. This is now fixed; the H1 is always a clean title.
  • Web Clipper: long article accuracy Clipping long pages that mix footnotes, tables, code blocks, and images now produces cleaner Markdown with fewer structural breaks.
  • Mermaid: black-fill rendering bug Diagrams rendered as solid black in the enlarged viewer and archive preview. Fixed across all four themes and all diagram types.
  • Mermaid: font inconsistency in Japanese diagrams Diagrams containing Japanese text were using a mix of fonts. Font rendering is now uniform.
  • Git sync: write conflict guard Concurrent writes during sync could corrupt the stored state. A write-conflict guard now prevents this, and any sync interruption after conflict resolution recovers cleanly on the next operation.
  • Settings sync: field-level 3-way merge When settings differed across devices, sync previously picked one side wholesale. It now merges at the field level, preserving each device's intentional changes.
  • Custom export presets now sync via Git User-defined export presets were excluded from Git sync. They are now included, so presets created on one device appear on all others.
