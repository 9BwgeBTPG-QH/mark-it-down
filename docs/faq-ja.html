よくある質問

Q: Chromeのキャッシュを削除してもデータは消えませんか？

A: 消えません。 ノートデータは拡張機能専用ストレージ（chrome.storage.local）に保存されており、ブラウザのキャッシュとは別の場所です。

  • ブラウザキャッシュ削除 → データは保持 ✅
  • 拡張機能アンインストール → データは消去 ⚠️

Q: ノートが増えすぎて重くなることはありますか？

A: Mark It Downは「作業場」として設計されており、大量のデータを貯め込むことは想定していません。

推奨される使い方:

  • 作業中のノート20件程度を目安に
  • 完成したノートは Archive → Export → 削除
  • サイドバー下部のストレージゲージで使用量を確認できます（80%超で警告）
  • Git同期で外部バックアップも可能

💡 作業場の片付け: Mark It Downは一時的な作業場です。定期的に「卒業」させることで快適に使い続けられます。

Q: 起動が速いのはなぜですか？

A: 遅延ロードとコード分割により、初期ロードを86%削減しています。

Mark It Downは起動速度を最適化するため、以下の戦略を採用：

  • Mermaid遅延ロード - Mermaid図を含まないノートでは2.3MBのライブラリが全くロードされない
  • Code Splitting - 7つのチャンクに分割し、並列ロードとキャッシュを最適化
  • Chrome専用ストレージ - ファイルシステムアクセス不要、即座にデータ取得

Q: 大きなファイル（100KB+）でエディタが重くなりませんか？

A: 最適化により、大規模ドキュメントでも快適に編集できます。

Mark It Downは大規模ドキュメント（100KB+、150+コードブロック）向けに以下の最適化を実装しています：

  • CodeMirror無効化 - 100KB超で自動無効化（読み込み時間57%短縮：7秒→3秒）
  • ローディングスケルトン - 100KB+の初期読み込み時にスピナー + スケルトンUI表示（UX向上）
  • CSS content-visibility - 画面外コンテンツのレンダリングをスキップ
  • CSS containment - reflow/repaint範囲をエディタ内に限定
  • デコレーション最適化 - 変更範囲のみ再計算（インクリメンタル更新）
  • 自動spellcheck無効化 - 100KB超のドキュメントでCPU負荷を軽減

注意点： 100KB超のドキュメントではコードブロックのシンタックスハイライトが無効化されます。LaTeX数式も100KB超では無効化されます（Milkdown Crepeの依存関係による）。

💡 設計思想: Mark It Downは「咀嚼の場所」です。長大な生データの保管場所ではなく、整理・編集して「卒業」させる場所として使用してください。

Q: ストレージ容量はPCの何に依存しますか？

A: 主にディスク空き容量に依存します。

使用パターン ディスク使用量 RAM使用量 起動速度
通常使用（20-50ノート） 数MB 数MB 一瞬
想定外の使用（1,000ノート超） - - 遅くなる可能性あり → Archive/Exportで片付けを推奨

💡 快適に使うために: 大量のノートを保存するのではなく、「咀嚼 → 片付け → 卒業」のサイクルを回しましょう。
