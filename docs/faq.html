Q: Will my data be deleted if I clear Chrome's cache?

A: No, your data will not be deleted. Notes are stored in Chrome extension storage (chrome.storage.local), which is separate from browser cache.

Q: Will the app slow down if I have too many notes?

A: Mark It Down is designed as a "workspace," not a storage system. It's not intended for storing large amounts of data.

Recommended usage:

  • Keep around 20 active notes
  • Export completed notes: Archive ‚Üí Export ‚Üí Delete
  • Check storage usage in the sidebar (warning at 80%+)
  • Use Git sync for external backup

üí° Workspace cleanup: Mark It Down is a temporary workspace. Regularly "graduate" your notes to keep it running smoothly.

Q: Why is the app so fast to start?

A: Lazy loading and code splitting reduce initial load by 86%.

Mark It Down optimizes startup speed with the following strategies:

  • Mermaid lazy loading - The 2.3MB library doesn't load at all for notes without Mermaid diagrams
  • Code splitting - Split into 7 chunks for parallel loading and caching
  • Chrome extension storage - No file system access needed, instant data retrieval

Q: Will the editor slow down with large files (100KB+)?

A: Optimizations ensure smooth editing even for large documents.

Mark It Down implements the following optimizations for large documents (100KB+, 150+ code blocks):

  • CodeMirror disabled - Automatically disabled for 100KB+ files (57% faster loading: 7s ‚Üí 3s)
  • Loading skeleton - Spinner + skeleton UI during initial load for 100KB+ documents
  • CSS content-visibility - Skips rendering content outside the viewport
  • CSS containment - Limits reflow/repaint range to the editor
  • Decoration optimization - Recalculates only changed ranges (incremental updates)
  • Auto spellcheck disabled - Reduces CPU load for 100KB+ documents

Note: For 100KB+ documents, code block syntax highlighting and LaTeX equations are disabled. This is an intentional trade-off prioritizing loading speed.

üí° Design philosophy: Mark It Down is a "place to digest." It's not for storing large raw data, but for organizing and editing before "graduating" your notes.
