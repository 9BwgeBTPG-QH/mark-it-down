Mark It Down

Features

Introduction

Write in Markdown.

No AI. No plugins. Just one pane.
Write, see instantly.

Two Modes

New Tab — Full screen

Dedicated workspace. Zen mode available (F11).

Side Panel — Beside the web

Write alongside AI. Write while you browse.

Sticky toolbar

Toolbar stays visible while scrolling in Side Panel. Always accessible for quick actions.

Zen mode (New Tab only)

F11 or Ctrl+Shift+Z to hide all UI. Focus on writing. Escape to exit.

Focus Mode v1.6.0 Under Review

Alt+F to toggle. Auto-hide UI after 3 seconds. Typewriter scroll keeps cursor centered. Text dimming for focus zone.

Open both. No sync.

See the old version while editing the new one. Independent windows share the same data.

Markdown

GitHub Flavored Markdown

Tables, task lists, code blocks with syntax highlighting.

Mermaid diagrams

Flowcharts, sequence diagrams, pie charts. Click to zoom.

Math (LaTeX)

Inline $E=mc^2$ and block $$...$$ equations.

Callouts

GitHub Alerts: NOTE, TIP, IMPORTANT, WARNING, CAUTION. Color-coded for light/dark themes.

Page links

Internal links: [Text](#heading) for page navigation. GitHub-compatible heading ID generation.

Images

External URLs only (![alt](https://...)). Direct embedding not supported due to storage limits.

View

Table of Contents

Auto-generated TOC. Highlights current section while scrolling. Click to jump. TOC navigation in conflict resolution.

v1.6.0: Collapsible headings (20+), search bar (15+), keyboard navigation Under Review

Dark/Light/System theme

Switch themes in Settings. System theme follows OS preference.

Smooth transitions

200ms fade effect when switching notes. Scrollable code blocks (max-height: 400px for 20+ lines).

Auto Pair

Brackets, quotes, backticks auto-complete. Toggle in Settings.

Code block height limit

Long code blocks (20+ lines) are scrollable. Copy button copies full content from Markdown source.

Note Management

Five folders. Fixed.

Inbox (working) / Template (reusable) / Archive (locked) / Trash (pending) / System (read-only guides). Custom folders not supported by design.

Templates & Pin

Store templates in Template folder. Pin important notes to the top of any folder (★ button). Alt+P shortcut.

Drag & drop

Move notes between folders. Drop on folder area (folder name + note list).

Sort & search

Name or date order (clock icon). Incremental search in sidebar (1 character = instant filter). Matching folders auto-expand.

Duplicate detection

Warning banner for same-title notes. Side-by-side comparison to choose which to keep.

Sidebar resize

Drag to adjust width. Double-click to reset. Slack-style interaction.

Storage

Auto-save

No save button. Auto-saves after 0.5 seconds of typing. Debounced to prevent race conditions.

Large documents

100KB+ Markdown files. CodeMirror disabled automatically. Loading skeleton for UX. CSS optimizations for smooth editing.

v1.6.0: Archive lightweight renderer (70% faster) Under Review

Why Archive is faster

Notes in Archive folder use a lightweight static HTML renderer instead of the full ProseMirror editor. Since Archive notes are read-only, editing features (undo/redo, input monitoring) are skipped, resulting in 70% faster loading (3+ seconds → under 1 second for 100KB+ documents).

Chrome extension storage

Survives browser cache clear. Data persists until extension uninstall. Unlimited storage permission.

Import/Export

Drag & drop .md/.zip files. Export: single .md, all notes .zip, or Archive pinned .zip.

Backup

Export all as .zip or sync to GitHub/GitLab. Git sync for version history.

Archive Pinned export

Export only pinned Archive notes as a .zip file. Pin important completed notes in Archive folder, then use "Export → .zip (Archive Pinned)" to back them up all at once. Useful for regular backups of important completed work.

Failsafe deletion

Synced notes: Trash → Sync → Delete (3 steps). Unsynced notes: Trash → Delete (2 steps). Sync status badges in Trash.

Storage repair & analysis

Settings → Troubleshooting: Repair Storage (duplicate detection), Storage Analysis (usage stats, bulk archive).

Git Sync

GitHub / GitLab

Connect with Personal Access Token (Fine-grained PAT recommended). AES-GCM encrypted storage.

Intentional version control

Not automatic sync like OneDrive. Manual commits = meaningful snapshots. Local auto-save (no history) vs remote commits (full history).

Commit & Sync

True synchronization: Commit → Pull → Resolve conflicts → Push. Custom commit messages. Batch commits (1 commit for all changes).

Pull & merge

Resolve conflicts side-by-side. Risk level indicators (Low/Medium/High/Critical). Myers diff algorithm. TOC navigation. Scroll sync.

Git Fetch

Check remote status without changing local. Auto-fetch on startup. Banner shows remote updates.

Diff & history

Compare versions side-by-side. View past 20 commits. Overview ruler for diff navigation.

File tracking

Auto-delete old files when moving folders or renaming. Prevents duplicate files on remote.

Discard Changes

Revert local changes to last push state. Git Settings → Danger Zone.

System notes

Excluded from Git sync (local only). Trash + Sync deletes from remote. Auto-cleanup on upgrade.

Dual Mode

Open New Tab and Side Panel simultaneously.

Independent windows

Edit in one. Read in the other.

Conflict resolution

Merge Changes button blinks when both windows edit same note. Choose which to keep. Same UI for Git Pull conflicts.

Comparison tools

Sync scroll, diff highlight (+green/-red), Overview Ruler (click to jump), TOC navigation, quick scroll buttons.

Responsive conflict UI

Side Panel/narrow screens: single pane view with large buttons. Keyboard navigation (Arrow keys, Escape).
